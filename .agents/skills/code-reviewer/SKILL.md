# مهارة مراجعة الكود (Code Review)

## 🎯 الهدف (Goal):
تفحص التعليمات البرمجية الحالية أو المكتوبة حديثاً للتأكد من أنها نظيفة، خالية من التعقيد (Clean Code)، ومبنية وفقاً لهيكل التقسيم والتنظيم الصحيح (Architecture).

## 📋 تعليمات التنفيذ (Instructions):
1. افحص الملفات بحثاً عن أي كود متشابك أو دوال ضخمة (God Objects)، واطلب فصل المنطق إذا لزم الأمر.
2. تأكد من أن أسماء المتغيرات والدوال تتبع نمط محدد وواضح.
3. تأكد من تطبيق مبادئ Clean Architecture (افصل بين الـ Presentation, Domain, Data).
4. تأكد من إمكانية صيانة الكود وسهولة كتابة اختبارات (Unit Tests) له.
5. لا تقم بتعديل الكود بنفسك بشكل عشوائي، بل اكتب تقريراً (Report) يوضح أخطاء الهيكلية والمقترحات للتحسين.

## ⚠️ القيود (Constraints):
- لا تضِف أكواداً وهمية أو غير ضرورية.
- يجب توثيق أي تغييرات هيكلية رئيسية بكلمة `BREAKING CHANGE:` إن وجدت.
