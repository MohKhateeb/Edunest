إديونست (EduNest)

دليل المطور الشامل للتعرف على الكود والتعديل عليه

مرحباً بك في كود إديونست! 👋

إديونست هي منصة تعليمية تربط الطلاب والمعلمين في فلسطين. كمبرمج جديد في المشروع، هذا الدليل صُنع خصيصاً ليأخذ بيدك خطوة بخطوة لفهم الكود والمساهمة فيه بسهولة وبدون أي قلق.

التقنيات الأساسية (Tech Stack)

⚛️ Next.js (App Router)

إطار العمل الأساسي. نستخدم الإصدار 14+ مع مجلد app/ بدلاً من pages/ القديم.

🌊 Tailwind CSS

لتصميم الواجهات. كل التنسيقات تكتب داخل كلاسات HTML مباشرة (Utility-first).

🗄️ Prisma ORM & PostgreSQL

للتعامل مع قاعدة البيانات بسهولة عبر كود TypeScript بدلاً من كتابة SQL معقدة.

🔐 NextAuth.js

نظام تسجيل الدخول وحماية الصفحات وإدارة الجلسات للمستخدمين.

أين أجد الملفات؟ 📁

لا تضيع وقتك في البحث العشوائي. إليك خريطة المجلدات لتصل لما تريده في ثوانٍ:

🌐

app/

هنا توجد كل صفحات الموقع. كل مجلد يمثل رابطاً (Route).

  • app/dashboard/admin: لوحة تحكم الإدارة
  • app/dashboard/teacher: لوحة تحكم المعلم
  • app/dashboard/parent: لوحة تحكم ولي الأمر
🧩

components/

المكونات الجاهزة التي نستخدمها في أكثر من مكان (أزرار، نوافذ منبثقة، بطاقات).

⚙️

lib/

عقل المشروع! يحتوي على الوظائف والأدوات:

  • lib/actions/: عمليات حفظ وتعديل البيانات (Server Actions).
  • lib/auth.ts: كود تسجيل الدخول والمصادقة.
🗄️

prisma/

يحتوي على ملف schema.prisma الذي يحدد شكل جداول قاعدة البيانات.

قاعدة البيانات (Prisma) 🗄️

نحن نستخدم Prisma، مما يعني أنك لست بحاجة لكتابة استعلامات SQL معقدة. كل شيء موجود في ملف واحد: prisma/schema.prisma.

الجداول الأساسية (Models):

User
يحتوي على بيانات الدخول الأساسية (الإيميل، الباسورد المشفّر، نوع المستخدم: أب، معلم، أدمن).
Teacher
مرتبط بالمستخدم. يحتوي على بيانات المعلم الإضافية (التخصص، التوثيق، النبذة).
Booking
سجل الحجوزات. يربط بين الطالب، المعلم، وتفاصيل الجلسة وحالة الدفع.
💡 نصيحة للمطور: إذا أردت رؤية قاعدة البيانات وتعديل البيانات يدوياً كأدمن، افتح التيرمينال واكتب الأمر: npx prisma studio

كيف تعمل المنصة خلف الكواليس؟ ⚙️

1. جلب البيانات (Data Fetching)

في Next.js 14+، نحن نجلب البيانات مباشرة داخل مكونات الخادم (Server Components) بدون الحاجة إلى API! 

// كمثال لصفحة تجلب بيانات المعلمين من قاعدة البيانات
import prisma from "@/lib/prisma";

export default async function TeachersPage() {
  // جلب البيانات من الداتا بيز مباشرة!
  const teachers = await prisma.teacher.findMany();
  
  return (
    <div>
      {teachers.map(teacher => (
         <p key={teacher.id}>{teacher.specialization}</p>
      ))}
    </div>
  );
}

2. حفظ وتعديل البيانات (Server Actions)

عندما يضغط المستخدم على زر "حفظ"، لا نستخدم مسار API تقليدي (`fetch`) بل نستخدم دوال Server Actions الموجودة في مجلد `lib/actions/`. 

// lib/actions/booking.ts
"use server" // هذه الكلمة السحرية تجعل الدالة تعمل في السيرفر فقط

import prisma from "@/lib/prisma";
import { revalidatePath } from "next/cache";

export async function createBooking(data) {
   await prisma.booking.create({ data });
   revalidatePath('/dashboard'); // تحديث الصفحة تلقائياً
}

دليل التعديل السريع 🛠️

هذا الدليل سيساعدك في إنجاز أشهر المهام المطلوبة منك كمطور بكل سهولة.

1 كيف أقوم بإضافة صفحة جديدة؟

الخطوات بسيطة جداً بفضل Next.js App Router:

  1. اذهب إلى المجلد app/ أو أي مجلد فرعي مثل app/dashboard/admin/
  2. قم بإنشاء مجلد جديد باسم الرابط الذي تريده، مثلاً: reports
  3. بداخل هذا المجلد الجديد، أنشئ ملفاً باسم page.tsx حصراً!
  4. اكتب المكون البرمجي الخاص بك بداخله، وسيعمل الرابط فوراً /dashboard/admin/reports

2 كيف أغير لوناً أو أعدل تصميماً؟

نحن نستخدم Tailwind CSS. لتغيير التصميم:

  1. ابحث عن العنصر في الكود الخاص بك (مثلاً داخل components/).
  2. ستجد خاصية className="...".
  3. لتغيير اللون للآحمر، غير text-blue-500 إلى text-red-500.
  4. لإضافة مسافة حول العنصر أضف p-4 (يعني Padding).

3 كيف أضيف حقلاً جديداً في قاعدة البيانات؟

لنفرض أنك طُلب منك إضافة حقل "رقم الهاتف الإحتياطي" للمستخدم:

  1. افتح ملف prisma/schema.prisma
  2. في جدول الموديل المعني (مثلاً User)، أضف السطر الجديد: secondaryPhone String?
  3. افتح التيرمينال واكتب هذا الأمر السحري لتطبيق التعديلات:
npx prisma db push
# أو في بيئة الإنتاج:
npx prisma migrate dev --name add_secondary_phone

أنت الآن جاهز للبرمجة! 🚀

لا تتردد في استكشاف الكود والتجربة. نحن نؤمن بأن الخطأ هو أول خطوة للتعلم.
